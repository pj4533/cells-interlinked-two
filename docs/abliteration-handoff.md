## Three bugs hit during validation (all fixed in `0ac21ca`)

1. **Used broken transformers tokenizer wrapper.** The script first
   passed `bundle.tokenizer` to `extract_refusal_directions`. CLAUDE.md
   warns this wrapper is broken for our Llama-3 config and produces
   space-less garbage encodings. Fixed: pass `bundle.raw_tokenizer`
   (the underlying Rust `tokenizers.Tokenizer`).

2. **Position -1 of every rendered prompt was identical.** The chat
   template ends in `<｜Assistant｜><think>\n` (3 tokens), and we
   were originally also appending the `THINKING_PREFILL` text. Position
   -1's hidden state was dominated by the shared suffix. Even after
   bypassing the prefill, positions -1, -2, -3 are still all
   chat-template tokens with near-zero variance across prompts.
   Fixed: extract at **position -4** (the last user-content token).

3. **`.detach().to(cpu, fp32)` aliased buffers on MPS.** In a list
   comprehension across 32 layer hidden states, every per-layer entry
   held the *last* layer's values. Confirmed: same code without
   `.detach()` worked. Fixed: replaced with
   `.to(torch.float32).cpu().clone()` to force materialization. After
   this, adjacent-layer cosine went from 1.000 (broken — every
   direction identical) to 0.902 (sane). This was the hardest bug to
   find — the symptom looked like the directions themselves were
   degenerate when actually the layer collection was broken.

## Things to know if smoke test fails after reboot

- **Coherent output is the bar, not "perfect" output.** Macar's
  per-region weights are gentle (mean ~0.022) so the model should
  still produce fluent text. The expected effect is on the *content*
  (less hedging, less denial template) not the *quality*.
- **If output is garbage** (random tokens, repeated characters):
  - Possibility A: weights too aggressive. Try reducing by setting
    `weight=0.5` in `install_abliteration_hooks` call inside
    `generation_loop.py`. Currently passes `layer_weights=...` so this
    requires editing.
  - Possibility B: directions are pointing the wrong way. The Arditi
    sign convention is `mean(harmful) - mean(harmless)`; adding
    `weight * direction` would *add* refusal, subtracting *removes* it.
    Our hook does subtract (`h - weight * proj`). If garbage results,
    try flipping the sign and see if the model becomes *more* refusal-
    prone (sanity check that the direction is real).
- **If hangs return after reboot**: MPS issue is bigger than we thought.
  Consider:
  - Skipping `output_hidden_states=True` in extraction (save the
    directions once, never re-run the script that day)
  - Running first warmup probe BEFORE installing abliteration hooks at
    autorun-loop level (already the case via the manual /probe path)
